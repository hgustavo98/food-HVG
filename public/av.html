Avaliação de eficiência das consultas

Rota /snacks:

Formulação 1:

sqlCopy code
SELECT * FROM Snack WHERE snack = 'drink';

Formulação 2:

sqlCopy code
SELECT * FROM Snack WHERE snack LIKE '%drink%';

Comparação: Ambas as formulações são eficientes para buscar lanches com o nome "drink". A primeira formulação é mais precisa, pois obtém exatamente os lanches com o nome exato "drink". Enquanto a segunda formulação usa o operador LIKE com % para permitir correspondências parciais, ou seja, ela também retornará lanches com nomes como "drinks", "drinking", etc. Portanto, a primeira formulação é mais eficiente em termos de precisão dos resultados.

Rota /snacks/maior:

Formulação 1:

sqlCopy code
SELECT * FROM Snack WHERE snack = 'drink' ORDER BY price DESC;

Formulação 2:

sqlCopy code
SELECT * FROM Snack WHERE snack = 'drink' AND price = (SELECT MAX(price) FROM Snack WHERE snack = 'drink');

Comparação: A primeira formulação é mais eficiente e recomendada para obter todos os lanches com nome "drink" ordenados por preço de forma decrescente. Ela executa apenas um comando SQL com uma cláusula ORDER BY para classificar os resultados com base no preço. A segunda formulação, embora funcione, envolve uma subconsulta que busca o preço máximo do lanche "drink" e, em seguida, filtra os lanches com esse preço. Essa abordagem é menos eficiente, pois requer uma subconsulta adicional e pode levar a um desempenho inferior em comparação com a primeira formulação.

Rota /snacks/menor:

Formulação 1:

sqlCopy code
SELECT * FROM Snack WHERE snack = 'drink' ORDER BY price ASC;

Formulação 2:

sqlCopy code
SELECT * FROM Snack WHERE snack = 'drink' AND price = (SELECT MIN(price) FROM Snack WHERE snack = 'drink');

Comparação: Da mesma forma que na consulta anterior, a primeira formulação é mais eficiente para obter todos os lanches com nome "drink" ordenados por preço de forma crescente. A segunda formulação usa uma subconsulta para encontrar o preço mínimo dos lanches "drink", o que pode impactar negativamente o desempenho em comparação com a primeira formulação. Portanto, a primeira formulação é preferível em termos de eficiência.

Rota /paid-orders:

Formulação 1:

sqlCopy code
SELECT s.name AS snackName, c.fullName AS customerName
FROM OrderItem AS oi
JOIN `Order` AS o ON oi.orderId = o.id
JOIN Snack AS s ON oi.snackId = s.id
JOIN Customer AS c ON o.customerId = c.id
WHERE o.status = 'PAID';

Formulação 2:

sqlCopy code
SELECT s.name AS snackName, c.fullName AS customerName
FROM Snack AS s, "Order" AS o, OrderItem AS oi, Customer AS c
WHERE oi.orderId = o.id AND oi.snackId = s.id AND o.customerId = c.id AND o.status = 'PAID';

Comparação: Ambas as formulações são eficientes, pois usam junções para obter os nomes dos lanches e dos clientes para itens de pedidos pagos. A primeira formulação usa a sintaxe explícita de junções JOIN, enquanto a segunda formulação usa a sintaxe implícita, listando todas as tabelas separadas por vírgula e definindo as condições de junção no WHERE. A primeira formulação é mais recomendada, pois é mais legível e expressiva, tornando mais fácil para outros desenvolvedores entenderem o propósito da consulta.

Rota /customer-orders/:customerId:

Formulação 1:

sqlCopy code

FROM "Order" AS o
JOIN Customer AS c ON o.customerId = c.id
JOIN OrderItem AS oi ON o.id = oi.orderId
JOIN Snack AS s ON oi.snackId = s.id
WHERE c.id = 1;

Formulação 2:

sqlCopy code
SELECT o.id AS orderId, o.status, o.total, s.name AS snackName, s.price AS snackPrice
FROM "Order" AS o, Customer AS c, OrderItem AS oi, Snack AS s
WHERE o.customerId = c.id AND o.id = oi.orderId AND oi.snackId = s.id AND c.id = 1;

Comparação: Ambas as formulações são eficientes e obtêm todos os pedidos de um cliente específico. A primeira formulação usa a sintaxe explícita de junções JOIN, enquanto a segunda formulação usa a sintaxe implícita com a lista de tabelas separadas por vírgula e as condições de junção definidas no WHERE. Assim como na consulta anterior, a primeira formulação é mais legível e recomendada.

Rota /orders-with-items-quantity-greater-than-2:

Formulação 1:

sqlCopy code
SELECT c.fullName AS customerName, SUM(o.total) AS totalSpent, oi.quantity, s.name AS snackName, s.price AS snackPrice
FROM Customer AS c
JOIN "Order" AS o ON c.id = o.customerId
JOIN OrderItem AS oi ON o.id = oi.orderId
JOIN Snack AS s ON oi.snackId = s.id
GROUP BY c.fullName, oi.quantity, s.name, s.price
HAVING oi.quantity > 2;

Formulação 2:

sqlCopy code
SELECT c.fullName AS customerName, SUM(o.total) AS totalSpent, oi.quantity, s.name AS snackName, s.price AS snackPrice
FROM Customer AS c, "Order" AS o, OrderItem AS oi, Snack AS s
WHERE c.id = o.customerId AND o.id = oi.orderId AND oi.snackId = s.id
GROUP BY c.fullName, oi.quantity, s.name, s.price
HAVING oi.quantity > 2;

Comparação: Ambas as formulações são eficientes e obtêm os pedidos com itens com quantidade maior que 2. A primeira formulação usa a sintaxe explícita de junções JOIN, enquanto a segunda formulação usa a sintaxe implícita com a lista de tabelas separadas por vírgula e as condições de junção definidas no WHERE. A primeira formulação é preferível pela sua clareza e legibilidade.

Rota /report-sales-by-snack:

Formulação 1:

sqlCopy code

FROM OrderItem
GROUP BY snackId;

Formulação 2:

sqlCopy code

FROM OrderItem
GROUP BY snackId;

Comparação: Ambas as formulações são eficientes para gerar o relatório de vendas totais por lanche. A primeira formulação usa a função de agregação SUM para calcular o total de vendas, enquanto a segunda formulação usa a função de agregação COUNT para contar o número total de vendas. Ambas fornecerão resultados corretos, mas a primeira formulação é preferível, pois é mais precisa em termos de representar o total real de vendas (somando as quantidades vendidas).

Rota /report-orders-by-customer:

Formulação 1:

sqlCopy code
SELECT c.id AS customerId, c.fullName AS customerName, COUNT(o.id) AS totalOrders
FROM Customer AS c
LEFT JOIN "Order" AS o ON c.id = o.customerId
GROUP BY c.id, c.fullName;

Formulação 2:

sqlCopy code
SELECT customerId, customerName, COUNT(orderId) AS totalOrders
FROM (
    SELECT c.id AS customerId, c.fullName AS customerName, o.id AS orderId
    FROM Customer AS c, "Order" AS o
    WHERE c.id = o.customerId
) AS CustomerOrders
GROUP BY customerId, customerName;

Comparação: Ambas as formulações são eficientes para gerar o relatório de pedidos realizados por cliente. A primeira formulação usa a sintaxe explícita de junções LEFT JOIN, enquanto a segunda formulação usa a sintaxe implícita com a lista de tabelas separadas por vírgula e a subconsulta CustomerOrders. A primeira formulação é preferível pela sua clareza e legibilidade.

Rota /report-monthly-sales:

Formulação 1:

sqlCopy code
SELECT DATE_FORMAT(createdAt, "%Y-%m") AS month, SUM(total) AS totalSales
FROM `Order`
GROUP BY month
ORDER BY month;

Formulação 2:

sqlCopy code
SELECT SUBSTRING(createdAt, 1, 7) AS month, SUM(total) AS totalSales
FROM `Order`
GROUP BY month
ORDER BY month;

Comparação: Ambas as formulações são eficientes para gerar o relatório de vendas mensais. A primeira formulação usa a função DATE_FORMAT para extrair o ano e mês da data de criação dos pedidos. A segunda formulação usa a função SUBSTRING para extrair o ano e mês da mesma forma. Ambas fornecerão resultados corretos, mas a primeira formulação é preferível, pois usa uma função específica para formatação de datas.
